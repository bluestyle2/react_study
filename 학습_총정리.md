## 🟢 Section 02 - JavaScript 심화

### Chapter 01: Falsy와 Truthy

**Falsy 값** (if문에서 거짓취급):

- `undefined`, `null`, `0`, `-0`, `NaN`, `""` (빈 문자열), `0n`

**Truthy 값** (if문에서 참취급):

- 위의 Falsy 값을 제외한 모든 값
- 예: `"hello"`, `123`, `[]`, `{}`, `() => {}`

### Chapter 02: 콜백 함수 (Callback Function)

```javascript
function orderFood(callback) {

orderFood((food) => {
  console.log(food); // "떡볶이"

```

- 함수를 인자로 받아서 나중에 실행하는 함수

### Chapter 06: 배열 조작 메서드 (6가지)

| 메서드              | 설명             | 반환값      | 원본변경 |
| ------------------- | ---------------- | ----------- | -------- |
| `push()`            | 뒤에 요소 추가   | 새 길이     | ✅       |
| `pop()`             | 뒤에서 요소 제거 | 제거된 요소 | ✅       |
| `shift()`           | 앞에서 요소 제거 | 제거된 요소 | ✅       |
| `unshift()`         | 앞에 요소 추가   | 새 길이     | ✅       |
| `slice(start, end)` | 범위 잘라내기    | 새 배열     | ❌       |
| `concat()`          | 배열 합치기      | 새 배열     | ❌       |

### Chapter 07: 배열 순회 메서드 (5가지)

| 메서드        | 설명             | 용도                |
| ------------- | ---------------- | ------------------- |
| `forEach()`   | 모든 요소 반복   | 각 요소마다 작업    |
| `includes()`  | 요소 존재 확인   | true/false 반환     |
| `indexOf()`   | 요소 위치 찾기   | 첫 번째 인덱스 반환 |
| `findIndex()` | 조건에 맞는 위치 | 콜백 함수 사용      |
| `find()`      | 조건에 맞는 요소 | 요소 자체 반환      |

```javascript
let arr = [1, 2, 3];
arr.forEach((item) => console.log(item)); // 1, 2, 3

let index = arr.indexOf(2); // 1
let found = arr.find((item) => item > 2); // 3
```

### Chapter 09: 배열 변형 메서드 (5가지)

| 메서드       | 설명                    | 예제                           |
| ------------ | ----------------------- | ------------------------------ |
| `filter()`   | 조건 맞는 요소만 필터링 | `arr.filter(item => item > 2)` |
| `map()`      | 모든 요소를 변형        | `arr.map(item => item * 2)`    |
| `sort()`     | 정렬 (원본 변경)        | `arr.sort((a,b) => a-b)`       |
| `toSorted()` | 정렬 (새 배열)          | `arr.toSorted()`               |
| `join()`     | 배열 → 문자열           | `arr.join("-")` → "1-2-3"      |

```javascript
let arr = [1, 2, 3];
let doubled = arr.map((item) => item * 2); // [2, 4, 6]
let filtered = arr.filter((item) => item > 1); // [2, 3]
```

### Chapter 12: 콜백 지옥 해결 (Promise)

```javascript
function add10(num) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (typeof num === "number") {
        resolve(num + 10);
      } else {
        reject("숫자가 아닙니다");
      }
    }, 2000);
  });
}

// Promise 체이닝
add10(0)
  .then((result) => {
    console.log(result); // 10
    return add10(result);
  })
  .then((result) => {
    console.log(result); // 20
  })
  .catch((error) => {
    console.log(error);
  });
```

**3가지 상태:**

- `pending` : 대기 중
- `fulfilled` : 성공 (resolve)
- `rejected` : 실패 (reject)

### Chapter 13: Async/Await

```javascript
async function getData() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ name: "오준혁", id: "winterlood" });

  });

async function printData() {

  console.log(data);

printData();
```

- `async` : 함수를 비동기 함수로 만듦
- `await` : 비동기 작업 완료 대기 (async 함수 내부에서만 사용)

## 🟣 Section 06 - Props와 컴포넌트 분리

### 핵심: Props를 통한 데이터 전달

```javascript
function App() {
  const [count, setCount] = useState(0);

  const onClickButton = (value) => {
    setCount(count + value);
  };

  return (
    <div className="App">
      <Viewer count={count} />
      <Controller onClickButton={onClickButton} />
    </div>

// Props 받아서 사용
function Viewer({ count }) {
  return <h1>{count}</h1>;
}

function Controller({ onClickButton }) {
  return (

      <button onClick={() => onClickButton(-1)}>-</button>
    </>
  );
}
```

- 부모 → 자식 : Props로 데이터/함수 전달
- 자식 → 부모 : Props로 받은 함수 호출

## 💜 Section 07 - useEffect (생명주기)

### useEffect의 3가지 상황

```javascript
const [count, setCount] = useState(0);
const [input, setInput] = useState("");
const isMount = useRef(false);

// 1. 마운트 (컴포넌트 생성)
useEffect(() => {
  console.log("마운트됨");
}, []); // 의존성 배열이 빈 배열

// 2. 업데이트 (특정 상태 변화)
useEffect(() => {
  console.log("count 또는 input 변경됨");
}, [count, input]); // 의존성 배열에 상태명 포함

// 3. 언마운트 (컴포넌트 제거) - 정리함수
useEffect(() => {
  return () => {
    console.log("언마운트됨");
  };
}, []);

// 4. 맨 처음 업데이트만 제외하기
useEffect(() => {
  if (!isMount.current) {
    isMount.current = true;
    return;
  }
  console.log("처음 외에만 실행");
}, [count, input]);
```

**의존성 배열 (dependency array)**

- `[]` : 1번만 실행 (마운트)
- `[state]` : state 변할 때마다 실행
- 생략 : 리렌더링될 때마다 실행

## 🖤 Section 08 - Todo List 만들기 (기초)

### 구조

```
App
├─ Header (제목)
├─ Editor (새 todo 입력)
├─ List (todo 목록)
│  └─ TodoItem (개별 todo)
```

### 핵심 로직

```javascript
function App() {
  const [todos, setTodos] = useState(mockData);
  const idRef = useRef(3);

  // 1. 생성
  const onCreate = (content) => {
    const newTodo = {
      id: idRef.current++,
      isDone: false,
      content: content,
      date: new Date().getTime(),
    };
    setTodos([newTodo, ...todos]); // 앞에 추가
  };

  // 2. 업데이트 (완료 토글)
  const onUpdate = (targetId) => {
    setTodos(
      todos.map((todo) =>
        todo.id === targetId ? { ...todo, isDone: !todo.isDone } : todo
      )
    );
  };

  // 3. 삭제
  const onDelete = (targetId) => {
    setTodos(todos.filter((todo) => todo.id !== targetId));
  };

  return (
    <div>
      <Header />
      <Editor onCreate={onCreate} />
      <List todos={todos} onUpdate={onUpdate} onDelete={onDelete} />
    </div>
  );
}
```

**중요 메서드:**

- `map()` : 상태 수정 (변경된 새 배열)
- `filter()` : 항목 삭제 (필터링된 새 배열)

## ⚫ Section 09 - useReducer (상태 관리 고도화)

### useReducer란?

복잡한 상태 로직을 한 곳에서 관리

```javascript
// 1. Reducer 함수 정의
function reducer(state, action) {
  switch (action.type) {
    case "CREATE":
      return [action.data, ...state];
    case "UPDATE":
      return state.map((item) =>
        item.id === action.targetId ? { ...item, isDone: !item.isDone } : item
      );
    case "DELETE":
      return state.filter((item) => item.id !== action.data);
    default:
      return state;
  }
}

// 2. useReducer 사용
function App() {
  const [todos, dispatch] = useReducer(reducer, mockData);

  // 3. dispatch로 액션 전달
  const onCreate = (content) => {
    dispatch({
      type: "CREATE",
      data: newTodo,
    });
  };

  const onUpdate = (targetId) => {
    dispatch({
      type: "UPDATE",
      targetId: targetId,
    });
  };

  const onDelete = (targetId) => {
    dispatch({
      type: "DELETE",
      data: targetId,
    });
  };
}
```

**useState vs useReducer**

- `useState` : 간단한 상태 (숫자, 문자)
- `useReducer` : 복잡한 상태 (배열, 객체 여러 개)

## 🌟 Section 10 - useCallback (성능 최적화)

### useCallback이란?

함수를 메모이제이션 (캐싱)하여 불필요한 리렌더링 방지

```javascript
import { useCallback } from "react";

function App() {
  const [todos, dispatch] = useReducer(reducer, mockData);

  // useCallback 없음 → 렌더링될 때마다 새로운 함수 생성
  // const onCreate = (content) => {
  //   dispatch({ type: "CREATE", data: newTodo });
  // };

  // useCallback 사용 → 의존성 배열에 값이 변할 때만 함수 갱신
  const onCreate = useCallback((content) => {
    const newTodo = {
      id: idRef.current++,
      isDone: false,
      content: content,
      date: new Date().getTime(),
    };
    dispatch({
      type: "CREATE",
      data: newTodo,
    });
  }, []); // 의존성 배열이 비어있음 = 함수가 절대 변하지 않음

  const onUpdate = useCallback((targetId) => {
    dispatch({
      type: "UPDATE",
      targetId: targetId,
    });
  }, []);

  const onDelete = useCallback((targetId) => {
    dispatch({
      type: "DELETE",
      data: targetId,
    });
  }, []);

  return (
    <div>
      <Header />
      <Editor onCreate={onCreate} />
      <List todos={todos} onUpdate={onUpdate} onDelete={onDelete} />
    </div>
  );
}
```

**useCallback 문법**

```javascript
const memoizedFunction = useCallback(
  (arg) => {
    // 함수 로직
  },
  [dependency]
); // 의존성 배열
```

**성능 최적화 목적:**

- Props로 받은 함수가 변하면 자식 컴포넌트도 리렌더링됨
- useCallback으로 함수를 고정 → 자식 리렌더링 방지
